RAG Retrieval Methods

Hybrid + Reranker

Combines dense (FAISS) & sparse (BM25) retrieval via Reciprocal Rank Fusion, then uses an LLM to re-score the top documents for maximum relevance. Best overall quality.

Agentic (LangGraph)

A 6-node autonomous pipeline: analyze → decompose → retrieve → rerank → synthesize → quality-check with retry loop. Most thorough but slowest.

Hybrid RRF

Fuses dense & sparse results with Reciprocal Rank Fusion. Good balance of keyword & semantic matching without the extra LLM reranking cost.

Naive Dense (FAISS)

Pure vector similarity search using Azure OpenAI embeddings and FAISS index. Fast but may miss keyword-specific matches.

BM25 Sparse

Classic keyword-based retrieval (TF-IDF family). Excellent for exact term matches but lacks semantic understanding.

Scoring: Confidence is computed via LLM self-evaluation (quality_check node). For formal benchmarks, RAGAS (Retrieval Augmented Generation Assessment) framework is used — measuring Faithfulness, Answer Relevance, and Context Precision.
